# Issues Index

This directory stores open issue-input notes and resolved follow-up records that should survive individual implementation sessions.

## Canonical Entry Points

- [open/index.md](./open/index.md)
  historical problem notes and unresolved issue inputs
- [resolved/index.md](./resolved/index.md)
  issue notes that have been sufficiently addressed and moved out of the open set

## Notes

- An issue-input note is preserved because it contains reusable lessons, not because it is a current spec.
- Current normative rules should live in canonical docs or active plans, not only in issue notes.
